Five-Minute Quick Start

From zero to a running test with HTML report.

1

Install UTO

terminal 
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/adrianpothuaud/UTO/main/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/adrianpothuaud/UTO/main/install.ps1 | iex
2

Scaffold a project

terminal 
uto init ./my-suite --template web
cd ./my-suite
3

Run the generated test

terminal 
uto run --target web
  UTO runner  →  phase: discovery
  Found 1 test binary
  [ smoke_journey ] running...
  [ smoke_journey ] ✓ passed  (1.2s)
  Suite: 1 passed, 0 failed
4

Generate an HTML report

terminal 
uto report --html
5

Open the interactive UI

terminal 
uto ui --open

Test Script Examples

Real-world patterns using the uto-test intent API.

Login flow

tests/login.rs 
use uto_test::startNewSession;

#[tokio::test]
async fn login_and_reach_dashboard() -> uto_core::error::UtoResult<()> {
    let s = startNewSession("chrome").await?;
    s.goto("https://app.example.com/login").await?;
    s.fill_intent("Username", "alice@example.com").await?;
    s.fill_intent("Password", "correct-horse").await?;
    s.click_intent("Sign in").await?;
    s.assert_visible("Dashboard").await?;
    s.close().await
}

E-commerce checkout

tests/checkout.rs 
#[tokio::test]
async fn add_to_cart_and_checkout() -> UtoResult<()> {
    let s = startNewSession("chrome").await?;
    s.goto("https://shop.example.com/products").await?;

    // Works even if the button moves in the layout
    s.click_intent("Add to Cart").await?;
    s.assert_visible("Cart (1)").await?;

    s.click_intent("Proceed to Checkout").await?;
    s.fill_intent("Card number", "4242 4242 4242 4242").await?;
    s.click_intent("Place order").await?;
    s.assert_visible("Order confirmed").await?;
    s.close().await
}

Shared helper — same test logic for web and mobile

tests/cross_platform.rs 
use uto_test::startNewSession;
use uto_core::session::UtoSession;

/// Shared login assertion — runs on any platform
async fn verify_login(s: &dyn UtoSession) -> UtoResult<()> {
    s.assert_visible("Dashboard").await?;
    s.assert_visible("Welcome, Alice").await
}

#[tokio::test]
async fn web_login_succeeds() -> UtoResult<()> {
    let s = startNewSession("chrome").await?;
    s.goto("https://app.example.com").await?;
    s.fill_intent("Email", "alice@example.com").await?;
    s.fill_intent("Password", "pass").await?;
    s.click_intent("Sign in").await?;
    verify_login(&*s).await?;
    s.close().await
}

#[tokio::test]
async fn android_login_succeeds() -> UtoResult<()> {
    let s = startNewSession("android").await?;
    s.fill_intent("Email", "alice@example.com").await?;
    s.fill_intent("Password", "pass").await?;
    s.click_intent("Sign in").await?;
    verify_login(&*s).await?; // same assertion, different platform
    s.close().await
}

UTO Studio — Visual Test Authoring

Record interactions in a live browser session. UTO Studio generates production-ready Rust test code using the intent API — no selectors, no XPath.

1

Launch Studio

uto ui --studio --open
2

Click ⏺ Record

A browser session opens. Every click, fill, and navigation is captured.

3

Interact naturally

Use the app as a real user would. UTO recognizes elements visually — no selectors needed.

4

Stop and save

Click ⏹ Stop. Studio generates a complete Rust test function and saves it to your tests/ directory.

Studio REST API

MethodEndpointDescription
GET/api/studio/statusRecording state + step count
POST/api/studio/startBegin recording session
POST/api/studio/stopStop + receive generated code
POST/api/studio/stepAppend a recorded interaction

Example stop response

POST /api/studio/stop → 200 
{
  "recording": false,
  "step_count": 5,
  "generated_code": "#[tokio::test]\nasync fn login_flow() ..."
}

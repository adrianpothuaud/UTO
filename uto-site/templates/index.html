Five Pillars of the Revolution

Every layer is designed with a single goal: let you write tests that read like English.

🏗️

Zero-Config Infrastructure

Forget setup nightmares. UTO auto-discovers installed browsers and SDKs; if nothing is found it silently provisions a version-pinned driver into a local cache. OS-level process groups guarantee no zombie processes are ever left behind.

✓ Complete
👁️

The Recognition Loop

UTO perceives elements visually using an embedded ONNX ML model, then anchors its guess against the Accessibility Tree via a Weighted Consensus algorithm. High-DPI coordinate normalization is built in.

⚙ In Progress
🧑

Human-Centric API

Tests are written in the language of user intent. .select(), .fill(), .should_be_visible() — if a human can describe the action, UTO can perform it.

◷ Planned
⚡

Hybrid Orchestrator

A high-performance Rust backbone using gRPC for multi-device synchronisation and binary streams for real-time visual feedback. Web and Mobile are a single unified canvas.

◷ Planned
🧩

Simplicity by Default

UTO absorbs recurring automation friction into reusable defaults: iframe targeting on web pages, scroll/fling traversal on long mobile screens, and safer handling for stale element or context transitions.

⚙ In Progress

Why UTO?

Current tools fail because they act like machines looking at code rather than humans looking at an interface.

  • Selenium / Appium — slowed by outdated request-response protocols and separate tool chains for web vs. mobile.
  • Cypress / Playwright — limited to web-only or browser-isolated kernels; no unified cross-platform story.
  • UTO — treats Web and Mobile as a single canvas, compiled to machine code, zero JVM or Node.js overhead.

Roadmap

Phase 1 & 2

Zero-Config + Driver Communication

Chrome / Android SDK discovery, ChromeDriver provisioning, WebSession and MobileSession via W3C WebDriver.

Phase 3

AI Recognition Loop

ONNX ML model foundation, visual element detection, Weighted Consensus resolver with Accessibility Tree anchoring, intent-style session API.

Phase 4

Framework Maturity & Reporting

First-class CLI lifecycle (init, run, report), structured JSON/HTML reporting, mobile parity hardening, and contributor onboarding.

Phase 5 — Next

UI Mode

Interactive browser-based test runner and debugger (uto ui). Real-time event stream, test tree, watch mode, screenshot timeline, and report replay — platform-agnostic across web and mobile.

Phase 6

Vision Model Integration

Real-world pre-trained UI detection model, recognition accuracy benchmarking, and model versioning strategy.

Phase 7+

Advanced Intent, CI/CD & Ecosystem

Semantic intent chaining, GitHub Actions / GitLab CI integration, report aggregation, trend dashboards, and community plugin API.

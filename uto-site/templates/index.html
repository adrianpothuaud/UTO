Four Pillars of the Revolution

Every layer is designed with a single goal: let you write tests that read like English.

🏗️

Zero-Config Infrastructure

Forget setup nightmares. UTO auto-discovers installed browsers and SDKs; if nothing is found it silently provisions a version-pinned driver into a local cache. OS-level process groups guarantee no zombie processes are ever left behind.

✓ Complete
👁️

The Recognition Loop

UTO perceives elements visually using an embedded ONNX ML model, then anchors its guess against the Accessibility Tree via a Weighted Consensus algorithm. High-DPI coordinate normalization is built in.

⚙ In Progress
🧑

Human-Centric API

Tests are written in the language of user intent. .select(), .fill(), .should_be_visible() — if a human can describe the action, UTO can perform it.

◷ Planned
⚡

Hybrid Orchestrator

A high-performance Rust backbone using gRPC for multi-device synchronisation and binary streams for real-time visual feedback. Web and Mobile are a single unified canvas.

◷ Planned

Why UTO?

Current tools fail because they act like machines looking at code rather than humans looking at an interface.

  • Selenium / Appium — slowed by outdated request-response protocols and separate tool chains for web vs. mobile.
  • Cypress / Playwright — limited to web-only or browser-isolated kernels; no unified cross-platform story.
  • UTO — treats Web and Mobile as a single canvas, compiled to machine code, zero JVM or Node.js overhead.
